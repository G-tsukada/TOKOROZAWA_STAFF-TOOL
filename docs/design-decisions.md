# アクタス所沢 スタッフ管理ツール 設計決定事項

作成日: 2026-07-02  
対象: actus-staff プロジェクト

---

## 1. プロジェクト概要

| 項目 | 決定内容 |
|------|----------|
| 種別 | 完全新規 Web アプリ（既存 oneone-zakkaten とは別） |
| 目的 | 部下の月次実績・タスク進捗・MTGログを一画面で管理 |
| 利用者 | 店長（塚田）＋ スタッフ 10 名 |
| 管理単位 | 月次（画面上部で月切り替え） |

---

## 2. 技術スタック

| 項目 | 決定内容 |
|------|----------|
| フレームワーク | Next.js（App Router）+ TypeScript |
| UI | shadcn/ui + Tailwind CSS |
| 状態管理（v1） | Zustand（モックデータ） |
| DB | Vercel Postgres（Neon） |
| ORM | Prisma |
| デプロイ | Vercel |
| ソース管理 | GitHub |

---

## 3. 画面構成

### PC（4ペイン固定）

| Pane | 内容 |
|------|------|
| Pane 1 | スタッフ名 ＋ カテゴリ担当（FN/IA グループ表示） |
| Pane 2 | タスク一覧（月次、手入力で追加・削除） |
| Pane 3 | 実績サマリー ＋ タスク進捗 |
| Pane 4 | カテゴリ MTG ログ（VLOG 風タイムライン） |

### スマホ
タブ切替（スタッフ / タスク / 進捗 / MTG）

---

## 4. Pane 別仕様

### Pane 1：スタッフ
- FN（家具）と IA（雑貨）でグループ表示
- スタッフ名 ＋ カテゴリバッジ
- スタッフの追加・削除・カテゴリ変更が可能（v1 は未実装、後から追加）
- スタッフを選ぶと Pane 2〜4 が連動

### Pane 2：タスク
- 月次 × スタッフ単位で管理
- テキスト入力で追加（Enter または ＋ ボタン）
- 削除可能
- タスクは手入力のみ（CSV 取込なし）

### Pane 3：実績 ＋ 進捗

**実績サマリー（3 セクション）**

| セクション | 表示項目 |
|------------|----------|
| 個人実績 | 予算 / 実績 / 達成率 / 差異（単位：千円） |
| 接客件数 | 予算 / 実績 / 達成率 / 差異（単位：件） |
| カテゴリ実績 | 項目ごとに 予算 / 実績 / 達成率 / 差異（単位：千円） |

- 達成率 100%以上 → 緑、80%未満 → 赤
- 「手動入力」ボタンで任意タイミングで入力・更新可能
- カテゴリ実績は項目名を自由に追加（VP売上、TEX売上 など）

**タスク進捗**
- ステータス選択：未着手 / 進行中 / 完了 / 停滞
- メモ欄（自由記述）
- **停滞**のタスクのみ「店長コメント欄」が表示される
  - 閲覧：全員可
  - 編集：マネージャー PIN（4桁）を入力した場合のみ
  - 将来的に AI アドバイスに置き換え予定

### Pane 4：MTG ログ
- カテゴリ MTG の要約テキストをタイムライン表示（新しい順）
- 入力フロー：スマホ録音 → Gemini で要約 → アプリに貼り付け
- カテゴリで自動表示（VP MTG → VP 担当スタッフの Pane 4 に表示）
- 関連タスクを手動タグで紐づけ可能
- 担当なしスタッフは手動タグのみ

---

## 5. データ管理

### データ入力方法

| データ種別 | 入力方法 |
|------------|----------|
| 実績数値 | 手動入力（画面フォーム）または CSV アップロード（v1 実装済） |
| タスク | アプリ上で手入力 |
| MTG 要約 | スマホ録音 → Gemini → 貼り付け |
| 店長コメント | PIN 認証後に手入力（将来 AI 化） |

### CSV 仕様（実績取込）
- 共通列：スタッフ名、月、売上、目標、達成率
- カテゴリ別列：`categoryMetrics` の JSON に柔軟に格納
- v1 は手動入力が主、CSV は後から整備

---

## 6. 認証・権限

| 項目 | v1 の仕様 |
|------|-----------|
| ログイン | 共通パスワード |
| 編集権限 | 全員が全データ編集可 |
| 店長コメント欄 | マネージャー PIN（4桁）が必要 |
| v2 以降 | 個人別ログイン・役割別権限に拡張予定 |

---

## 7. DB スキーマ（方針）

| テーブル | 用途 |
|----------|------|
| `staff` | スタッフ名・チーム（FN/IA） |
| `categories` | カテゴリ（VP, TEX, DSB, CULTI, グリーン, ランドセル, ギフト） |
| `staff_categories` | スタッフとカテゴリの多対多（南澤さんの CULTI+グリーン に対応） |
| `performance_records` | 月次実績（売上・目標・接客件数・カテゴリ実績 JSON） |
| `tasks` | 月次タスク（ステータス・メモ） |
| `task_advice` | 停滞タスクへの店長コメント（別テーブルで将来 AI 化しやすく） |
| `mtg_logs` | MTG 要約テキスト（カテゴリ・日付） |
| `mtg_log_tasks` | MTG と タスクの紐づけ（手動タグ） |
